1. Origen del proyecto

El proyecto comenzó como "Gestion 0.0.2", una aplicacion web para gestion y logistica de camiones y rutas. El usuario (agus123e21) contaba con un repo anterior llamado BetaLogistica.0.0.1 (branch master). La idea principal era contar con una herramienta propia para administrar viajes de fletes: calcular rutas, asignar camiones, y llevar registro de cada envio.

3. Funcionalidades implementadas

3.1 Patron IIFE y encapsulamiento

Toda la logica de JavaScript se encuentra dentro de un IIFE (Immediately Invoked Function Expression) para evitar contaminacion del scope global.

3.2 Navegacion

  • Sidebar con botones para cada tab
  • Bottom nav optimizada para mobile
  • Tabs: Nueva Ruta, Mis Viajes, Mapa, Mis Camiones

3.3 PWA (Progressive Web App)

  • Service Worker v2 con cache offline-first
  • Manifest.json para instalacion como app
  • Cachea assets criticos incluyendo el GeoJSON de provincias

3.4 Geocodificacion

  • Api de Nominatim (OpenStreetMap) para resolver direcciones a coordenadas
  • Autocompletado de sugerencias mientras se escribe

3.5 Routing real

  • OSRM (gratuito) como servicio principal con steps=true para obtener nombres de calles
  • ORS como fallback (API key vacia, siempre cae a OSRM)
  • Haversine como ultimo recurso si no hay conexion
  • Parametros HGV para camiones (altura, ancho, largo, peso, carga por eje)

3.6 Mapas duales con Leaflet

  • Mapa inline en el formulario Nueva Ruta
  • Mapa full-screen en el tab Mapa
  • Tiles de CARTO dark_all (tema oscuro)
  • Restriccion geografica a Argentina con maxBounds
  • Overlay de 24 provincias argentinas con labels

3.7 CRUD completo de envios

  • Crear, editar, eliminar viajes
  • Estados: Pendiente, En Transito, Entregado
  • Almacenamiento en localStorage
  • Persistencia de datos entre sesiones

3.8 Sistema multi-camion

  • Array camiones[] con: nombre, patente, camionero, dimensiones, peso, consumo
  • Tab "Mis Camiones" con CRUD completo
  • Selector obligatorio de camion en "Nueva Ruta"
  • Cada envio almacena camionId
  • Consumo calculado segun el camion asignado

3.9 Filtros y busqueda

  • Filtrar por estado (Todos, Pendiente, En Ruta, Entregado)
  • Buscar por destino, carga, cliente, remito
  • Paginacion con lazy loading

3.10 Modal de datos del viaje

  • Selector de camion
  • Campos: cliente, N de remito
  • Aparece despues de calcular la ruta
  • Editable desde el detalle del viaje
